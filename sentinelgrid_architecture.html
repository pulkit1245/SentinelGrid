1. High-Level Architecture
SentinelGrid is a layered, event-driven system that fuses IoT/SCADA telemetry, permit-to-work (PTW) events, CCTV-derived vision events, and shift-roster data into a single live Plant Risk Graph. A synthetic simulator emits zone-level sensor readings, permit events, and shift transitions over WebSocket; the FastAPI backend ingests these, normalizes and time-aligns them, and writes them into an in-memory/Neo4j graph. A multi-agent reasoning layer (Compound Risk, Digital Permit, Incident-Pattern RAG, Compliance Audit) continuously evaluates the graph for dangerous combinations that no single threshold would catch, and an Orchestrator Agent arbitrates conflicting signals before triggering multi-channel alerts, evidence snapshots, and a drafted regulatory incident report. Every alert carries the graph-path that triggered it, so a safety officer can see why, not just that.

Authentication Flow
Safety officers, plant admins, and auditors log in via /api/v1/auth. The backend issues a short-lived JWT access token (15 min) plus a rotating refresh token (7 days), HS256-signed, secret in environment variables. The access token lives in memory in the React app (never localStorage); the refresh token sits in an HttpOnly cookie. Every protected route depends on a get_current_user dependency that validates the JWT and role (safety_officer / plant_admin / auditor).
Authorization & Security Architecture
Role-based access control gates evacuation-trigger and compliance-export endpoints to plant_admin only. All ingestion adapters (OPC-UA/Modbus/MQTT bridges) run in an isolated network segment and never expose write-back control to plant actuators in the hackathon scope — SentinelGrid is decision support, not autonomous control, and any evacuation action requires a human "confirm" click, logged to audit_logs. Evidence snapshots are written once and marked immutable (no UPDATE grant on that table) for post-incident forensic integrity. Secrets (Twilio, Mapbox, LLM API keys, DB credentials) are injected via environment variables, never committed.
Caching, Storage & Scalability
Redis caches the latest per-zone risk score and the last 60 minutes of rolling aggregates so the cockpit's polling/WebSocket layer never hits Postgres directly for live reads. Raw sensor time-series lives in TimescaleDB hypertables; transactional data (users, permits, zones, alerts) in PostgreSQL; CCTV frame snapshots and evidence exports in S3-compatible object storage; the Plant Risk Graph in Neo4j (networkx fallback for the hackathon demo). The non-invasive ingestion-adapter design means new plants/zones are added via configuration, not re-architecture — horizontal scaling is achieved by adding Kafka partitions per zone-group and stateless FastAPI/worker replicas behind a load balancer.

Plant Risk Graph (Neo4j) Ontology
Nodes: Zone, Sensor, Permit, Worker, Equipment, HazardClass, Incident (mined from historical reports). Edges: LOCATED_IN (Sensor/Worker/Equipment → Zone), OVERLAPS_WITH (Permit → Zone, time-bounded), PRECEDES (shift-boundary temporal edge), MAINTAINED_BY (Equipment → Worker), HISTORICALLY_CORRELATED_WITH (Zone/HazardClass → Incident, weight from RAG similarity score). Compound risk is detected as a graph-pattern match, e.g.: (:Permit {type:'hot_work'})-[:OVERLAPS_WITH]->(:Zone)<-[:LOCATED_IN]-(:Sensor {type:'gas'}) combined with a positive reading-slope and a shift boundary within 30 minutes.

6. AI, RAG, Knowledge Graph & Multi-Agent Architecture
Multi-Agent System
Four bounded, tool-using agents share the Plant Risk Graph as a common blackboard, coordinated by LangGraph:
  • Compound Risk Agent — runs Cypher/networkx pattern queries every ingestion cycle plus the XGBoost risk-scorer; outputs a 0–100 zone score.
  • Digital Permit Intelligence Agent — validates every active permit against live zone state at issuance and continuously during the work window; flags dangerous simultaneous operations.
  • Incident-Pattern RAG Agent — retrieves the most similar historical near-miss/incident and the governing regulatory clause for any elevated-risk zone.
  • Compliance Audit Agent — diffs live zone/permit state against curated OISD/DGMS/Factory Act rules and drafts corrective-action items.
  • Orchestrator Agent — arbitrates across all four agents' outputs, applies false-positive suppression (require ≥2 corroborating signals before "critical"), and decides escalation level.
Retrieval-Augmented Generation (RAG) Pipeline
Chunking: incident reports and regulatory clauses are split by section/paragraph (~300–500 tokens, 15% overlap) and tagged with source metadata (clause number, incident ID, date). Embedding: dense vectors stored in Qdrant; sparse BM25 index maintained in parallel. Retrieval: hybrid search (BM25 + dense, reciprocal-rank fusion), top-8 candidates. Re-ranking: cross-encoder re-ranks to top-3 by relevance to the query zone's current hazard signature. Context injection: only re-ranked, source-tagged chunks are injected into the LLM prompt — the LLM is instructed to never state a clause number it hasn't retrieved verbatim. Hallucination prevention: a citation-guard step post-checks every clause reference in the LLM's answer against the retrieved chunk set before it is shown to the user; unverified citations are stripped and the answer is regenerated once. Evaluation: a 10–15 clause gold-answer set is used to measure citation accuracy before every demo run; feedback (thumbs up/down on RAG answers) is logged for future re-ranker tuning.
Knowledge Graph Integration
The Incident-Pattern RAG Agent writes a HISTORICALLY_CORRELATED_WITH edge from the current Zone/HazardClass node to the matched Incident node whenever retrieval confidence exceeds a threshold, so future graph-pattern queries can directly traverse to precedent incidents without re-running retrieval — the graph becomes a growing, queryable index of "this pattern happened before."
Hard Rules-as-Floor (Safety Guarantee)
Statutory hard thresholds (e.g., gas ppm above the legal limit) are encoded as non-ML-overridable rules that always fire regardless of the ML/agent layer's output — the ML and agent stack only make the system faster and smarter above that floor, never the last line of defense for a life-safety trigger.

8. Data & Component Flow
Sequence — Compound Risk Detection
Simulator → /sensors/ingest → Enrichment Worker (time-align, rolling window) → Plant Risk Graph write (Zone node updated) → Compound Risk Agent (pattern query + XGBoost score) → Orchestrator Agent (corroboration check across Permit Agent + RAG Agent outputs) → if severity ≥ 'warning': Alert created (graph_path attached) → WebSocket push to Cockpit → human confirm (plant_admin) → Evidence Snapshot frozen → Twilio SMS dispatched → Compliance report drafted (LLM-filled template)
Module Dependency
frontend → backend API (REST + WS) → services layer → repositories → Postgres/Timescale; services layer also → agents package → graph client (Neo4j/networkx) and rag package (Qdrant + LangChain); cv package publishes independently into the same enrichment queue consumed by services. No module reaches into another member's package internals — only the shared JSON contracts in shared/schemas/ are relied upon, keeping the three members' work parallelizable.

12. Testing Plan
Unit Testing
Pytest for every service and repository (time-alignment math, permit-overlap logic, XGBoost feature builders, citation-guard string matching). Vitest + React Testing Library for cockpit component logic (heatmap color mapping, alert-list filtering, WebSocket reducer).
Integration Testing
Pytest + httpx against a test Postgres instance covering the full ingest→enrich→score→alert chain with a seeded scenario fixture, run in CI on every PR touching backend/ or agents/.
API Testing
Contract tests validating every response against the JSON schemas in shared/schemas/, so a frontend/backend field mismatch fails the build instead of surfacing at demo time.
AI Model Evaluation
Compound-risk scorer evaluated on a held-out labeled scenario set for precision/recall vs. the single-sensor baseline; Isolation Forest evaluated on drift-detection lead time.
Agent & RAG Evaluation
Orchestrator arbitration tested against scripted multi-signal scenarios (false-positive suppression check); RAG citation accuracy measured against the 10–15 clause gold-answer set (target: 100% on the curated demo corpus).
Knowledge Graph Validation
Cypher/networkx pattern-query unit tests confirming known compound-risk templates match on synthetic fixtures and do not match on negative (safe) fixtures.
Computer Vision Validation
YOLOv8/DeepSORT evaluated on a small labeled clip set for PPE-detection precision and ID-switch rate; false-positive PPE-violation rate tracked explicitly to avoid alarm fatigue.
Security Testing
JWT tampering and expired-token rejection tests; role-escalation attempt tests on the confirm/compliance-export endpoints; adversarial prompt-injection test set run through the citation-guard layer.
Performance Testing
Locust load test simulating concurrent zone updates to confirm the WebSocket/worker layer absorbs bursts without blocking the API, validating the horizontal-scaling claim in Section 1.
