Runs the graph-pattern query (hot-work permit ∩ zone ∩ gas-trend-slope-positive ∩ shift-boundary-within-30-min) every enrichment cycle, plus the XGBoost score once trained (Module 5).

Arbitrates across Compound Risk, Permit Intelligence, and (once available) Incident-RAG/Compliance agent outputs; requires ≥2 corroborating signals before escalating to "critical" (false-positive suppression); encodes the statutory hard-rules floor as a non-overridable first check.
agents/orchestrator_agent.py
5
Wire agent output to the Alerts API
On escalation, orchestrator calls Member 1's alert-creation service internally (not over HTTP — direct service call within the same backend process) with the full graph_path.

Module 2 — RAG Corpus & Embedding PipelineH8–H18
1
Curate the demo corpus
10–15 representative OISD/DGMS/Factory Act clauses plus mock/representative incident reports, kept small and clean specifically so citation accuracy can be honestly claimed at 100% for the demo.
rag/corpus/oisd_clauses/rag/corpus/incident_reports/
2
Build the chunking pipeline
Split by section/paragraph (~300–500 tokens, 15% overlap); tag every chunk with source metadata (clause number, incident ID, date).
rag/chunking.py
3
Generate and index embeddings in Qdrant
Batch-embed all chunks, upsert into a Qdrant collection with metadata payload for filtering by source type.
rag/embeddings.py
4
Build the parallel BM25 sparse index
Needed for hybrid retrieval so regulatory-jargon exact matches aren't lost to pure dense-embedding recall gaps.
5
Re-index script & corpus versioning
A single command to rebuild the index if the corpus changes, so last-minute clause additions before the demo don't require manual Qdrant surgery.
rag/scripts/reindex.py
Module 3 — Hybrid Retrieval, Reranking & Citation GuardH18–H26
1
Implement hybrid_retriever.py
BM25 + dense retrieval with reciprocal-rank fusion, returning the top-8 candidate chunks for a query.
rag/hybrid_retriever.py
2
Implement the cross-encoder reranker
Re-ranks the top-8 down to the top-3 most relevant to the query zone's current hazard signature.
rag/reranker.py
3
Build the LLM prompt for grounded answer generation
Injects only the reranked, source-tagged chunks; explicit instruction to never state a clause number not present in the retrieved context.
rag/prompts/answer_prompt.py
4
Build citation_guard.py
Post-checks every clause/incident reference in the LLM's answer against the retrieved chunk set; strips unverified citations and triggers one regeneration attempt before falling back to "not found in corpus."
rag/citation_guard.py
5
Expose POST /rag/query
Wraps the full retrieve → rerank → generate → guard pipeline behind a single authenticated endpoint the cockpit's RAG panel calls.
backend/app/api/v1/rag.py
6
Loading state & error handling in the query flow
Explicit timeout/error path so a slow LLM call degrades to a clear "retrieval succeeded, generation timed out" message rather than an opaque failure.
